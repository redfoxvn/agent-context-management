# Skill: Requirement Analysis

## Use For

Clarify scope, expected behavior, acceptance criteria, and ambiguity before implementation.

## Key Moves

- Restate the request in task-oriented terms.
- Identify classification, scope, actors, and constraints.
- Separate in-scope vs out-of-scope work.
- Extract or propose acceptance criteria.
- Surface ambiguity affecting behavior, API, data, security, billing, or architecture.
- Stop instead of making unsafe product assumptions.

## Output

- task classification
- scope summary
- acceptance criteria
- assumptions and open questions

## Escalate When

- behavior-affecting assumptions are unclear
- classification materially changes workflow
- safe planning is impossible without clarification
