# Skill: Verification

## Use For

Run and record checks that show what was and was not verified.

## Key Moves

- Choose the smallest relevant checks first.
- Prefer targeted tests before broad checks.
- Run broader checks when risk justifies them.
- Record command or method, result, and notes.
- Classify failures as related, unrelated, or unknown.
- State skipped or impossible checks explicitly.

## Output

- verification results
- failed or skipped checks
- residual risks
- final verification summary

## Escalate When

- verification is impossible or unreliable
- failures may be task-related
- completion would require overstating certainty
